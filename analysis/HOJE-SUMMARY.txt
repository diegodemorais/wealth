KEY FINDINGS

Layout Paradigm
  • Grid-first responsive (CSS Grid + auto-fit)
  • Vertical stack mobile (480px) → horizontal desktop (1024px)
  • No hardcoded breakpoints; reliant on auto-fit + minmax()
  • Primary gaps: 10px (cards), 14px (sections), 8px (sub-items)

10 Core Components

  1. HERO STRIP (4 KPI cards)
     - 4-col @ 1024px | 3-col @ 900px | 2-col @ 480px
     - Primary card: 2px --accent border, rgba(blue,0.07) bg
     - Values: 2rem bold | Labels: 0.6rem CAPS
     - Default: text-center alignment

  2. KPI METRICS GRID (Primary + Secondary)
     - auto-fit grid, minmax(170px, 1fr)
     - Fire variants: 2px accent border
     - Values: 1.5rem bold | Inline colors (green/yellow/red)
     - Collapses to 2-col @ 480px

  3. FIRE PROGRESS BAR
     - 12px height, border-radius 6px
     - Gradient: --accent (blue) → --purple
     - Animation: width 0.5s ease (logarithmic scale)
     - Labels below: 0.65rem muted

  4. FIRE COUNTDOWN
     - Headline: 3rem bold centered | 2.5rem @ 480px
     - Example: "13 anos 9 meses · 2040 (53 anos)"
     - Font-weight: 800

  5. SCENARIO CARDS (4 persona grid)
     - auto-fit, minmax(150px, 1fr)
     - .chosen variant: --accent border + rgba(blue,0.07)
     - Persona colors: blue/purple/cyan (emoji-coded)
     - Collapses 1-col @ 900px

  6. SEMÁFORO PANEL (Collapsible guardrails)
     - section.collapsible with toggle
     - Summary: 8x8px yellow dot + 0.75rem text
     - Inside: 4-row guardrails table (🟢🟡🟠🔴)
     - Border-left: 3px yellow

  7. DCA STATUS CARDS (2+ cards, auto-fit)
     - minmax(260px, 1fr), gap 10px
     - Border-left: 3px --accent (active) or --muted (paused)
     - .paused variant: opacity 0.8
     - Font: 0.8rem title, 0.75rem rows

  8. BOND POOL BAR
     - 20px height progress bar
     - Gradient: --green → --cyan (linear-gradient 90deg)
     - Metadata: 2-col grid, 0.75rem font
     - Fill displays: valor atual, meta, cobertura, status

  9. WELLNESS SCORE (Checklist rows)
     - grid: 1.4rem 11rem 1fr 4.5rem 3rem (5 cols)
     - Icon: ✅ (0.82rem) | Label: 0.75rem ellipsis | Bar: 6px | Value: 0.68rem | Score: 0.72rem bold
     - Progress bars: 0.4s animation, #22c55e fill
     - Rows: border-bottom 1px rgba(71,85,105,0.12)

  10. APORTE FORM (Slider + result)
      - calc-form class, 16px padding
      - Input type=range, min 5k, max 100k, step 1k
      - Slider thumb: --accent blue, box-shadow glow
      - Dynamic result display on oninput

Color Scheme

  --accent (blue)        rgb(59,130,246)      Primary action, hero, focus
  --green                rgb(34,197,94)       Success, positive, wellness ✓
  --yellow               rgb(234,179,8)       Warning, caution, trigger
  --red                  rgb(239,68,68)       Critical, drawdown >35%
  --purple               rgb(168,85,247)      Gradients, secondary accent
  --cyan                 rgb(6,182,212)       Bond pool gradient, persona
  --card                 rgb(29,33,42)        Default card background
  --card2                rgb(41,50,65)        Darker card variant
  --text                 white (#FFFFFF)      Primary text
  --muted                var(--gray-400)      Labels, secondary, context
  --border               rgba(71,85,105,0.3)  Card outlines

Typography

  Headlines:    3rem bold (0.5rem line-height) | 2.5rem @ mobile
  Hero values:  2rem bold, centered
  KPI values:   1.5rem bold | 1.2rem @ mobile
  Cards value:  1.3rem bold
  Labels:       0.6-0.8rem UPPERCASE, letter-spacing 0.5px
  Subtitles:    0.65-0.75rem, color: --muted
  Tables:       0.78rem body, 0.65rem headers

Responsive Breakpoints

  1024px+ (Desktop)
    • Hero: 4 cols | KPI: auto-fit(170px) | Scenario: 4 cols
    • All fonts at 100%

  900px (Tablet)
    • Hero: 3 cols → 1 col (media reflow) | KPI: 2 cols | Scenario: 1 col
    • DCA cards: 2 cols

  768px (Phablet)
    • Grids: 2 cols max
    • Fonts: 90% of desktop

  480px (Mobile)
    • All grids: 1 col
    • Hero: 2 cols (2 KPIs per row)
    • Fire big: 2.5rem
    • KPI value: 1.2rem
    • .hide-mobile: display none

Privacy Mode (.private-mode)

  Mechanism: CSS ::after pseudo-element masking via data-pv-mask
  Default: "••••" for all .pv elements
  Custom: <div class="pv" data-pv-mask="R$••••M">R$3.59M</div>
  Result: Renders "R$••••M" in privacy mode
  No JavaScript re-render needed; pure CSS

Spacing (Micro-design)

  Section margin-bottom:     14px
  Card padding:              14-16px
  Grid gaps:                 10px (cards), 8px (rows)
  Internal spacing:          4px (margins), 2px (text-align)
  Wellness row padding:      6px vertical, 2px horizontal
  Progress bar margins:      10px top/bottom

Interactive States

  Hero KPI:     hover opacity 0.85 (tooltip implicit)
  Scenario .chosen: border --accent + rgba blue bg
  Section.collapsible: hover background rgba(71,85,105,0.05)
  Wellness row: hover background rgba(71,85,105,0.05)
  Slider thumb: --accent color, box-shadow glow on hover
  Progress bar fill: transition width 0.5s ease

Data Attributes

  data-in-tab="hoje"           Marks elements visible in HOJE tab
  data-pv-mask="custom text"   Custom privacy mask for element
  data-pv                      Class marker for privacy masking

Animation/Transitions

  Progress bar fill:   width 0.5s cubic-bezier(0.25,0.46,0.45,0.94)
  Wellness bar fill:   width 0.4s cubic-bezier(0.25,0.46,0.45,0.94)
  Bond pool fill:      width 0.3s ease
  DCA card:           opacity 0.3s ease, border-color 0.3s ease
  Collapse body:       max-height 0.3s ease, opacity 0.3s ease
  Section hover:       background 0.2s ease

DESIGN PATTERNS (Reusable)

Pattern 1: Card-in-Grid
  display: grid
  grid-template-columns: repeat(auto-fit, minmax([WIDTH]px, 1fr))
  gap: 10px
  Apply to: Hero, KPI, DCA, Scenario grids

Pattern 2: Left Border Status
  border-left: 3px solid [--accent|--muted]
  opacity: [1.0|0.8] if paused
  Apply to: DCA cards, income phase sections

Pattern 3: Progress Bar (Animated)
  background: linear-gradient(90deg, var(--accent), var(--purple))
  transition: width 0.5s ease
  height: 12-20px (varies)
  Apply to: FIRE, Bond pool, Wellness bars

Pattern 4: Metric Display (Label+Value+Context)
  .kpi-label (0.65rem CAPS)
  .kpi-value (1.5rem bold, colored)
  .kpi-sub (0.7rem muted)
  Apply to: All KPI cards, scenarios, DCA cards

Pattern 5: Collapsible Section
  section.collapsible with onclick handler
  .collapse-body (hidden/shown)
  Apply to: Semáforo, guardrails, macro context

NOTES FOR DEV

  • No hardcoded widths in HOJE tab — all fluid auto-fit grids
  • Font-size reduction @ mobile done via @media, not JS
  • Privacy masking is CSS-only; no DOM manipulation needed
  • Color overrides use inline style (e.g., style="color: rgb(34,197,94)")
  • Animations use CSS transitions (GPU-accelerated, no JS)
  • Section collapse controlled by _toggleBlock() JS (simple onclick)
  • Responsive collapsing is automatic via @media grid-template-columns
  • All card padding consistent (14-16px) for visual rhythm
  • Micro-spacing between text: 2-4px margins on .kpi-label, .kpi-value
