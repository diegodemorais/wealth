## Limitações de Dados e Metodologia

### Por que não temos retorno Q1 completo?

1. **Sem histórico de preços pré-23/Mar**: O sistema de tracking foi criado em 2026-03-18. Não temos:
   - Preços de entrada para cada ETF no início de janeiro
   - Histórico de cash flows (aportes) dentro do Q1
   - Posições intermediárias

2. **Dados de benchmark limitados**:
   - MSCI World: só até 27/Feb (não há dados 28/Feb–31/Mar)
   - VWRA.L: preço final 23/Mar, mas preço inicial não disponível via web search
   - Câmbio: estimado para início de jan via spread bursátil

3. **Cronologia dos aportes**: O aporte de mar/2026 (R$56.485) ocorreu nos dias 5 e 11/Mar. Sem timestamps precisos, não é possível calcular TWR (time-weighted return) com exatidão.

### Abordagem para este report:

Documentar o **framework de cálculo** que será usado nos trimestres futuros, quando teremos histórico completo de preços dia a dia.

### Framework Aprovado para Q2+

A partir do Q2 2026 (01/Abr onwards), este framework será usado:

```
1. Retorno Total (BRL):
   - TWR (time-weighted return) usando preços diários de cada ETF
   - Ajuste para aportes via método de Dietz modificado

2. Retorno por Bucket:
   - Equity (SWRD + AVGS + AVEM + JPGL): vs MSCI World USD
   - Renda Fixa (IPCA+ 2029, 2040): vs IPCA+ index
   - Tático (Renda+ 2065, HODL11): vs Selic + BTC

3. Alpha:
   - Carteira equity vs VWRA.L (ou MSCI World USD-adjusted)
   - Controle por fator (size, value, quality)

4. Custos:
   - TER ponderado (diário, baseado em posições)
   - Spread cambial por operação
   - IR realizado por venda

5. Reconciliação:
   - Patrimônio início Q + aportes − saques = patrimônio fim Q (antes MtM)
   - Ganho MtM explicado por retorno + custos
```
